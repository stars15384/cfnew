## Project Overview

CFnew is a Cloudflare Worker/Pages proxy tool (v2.9.8c) supporting VLESS, Trojan, and XHTTP protocols. It runs entirely on Cloudflare's edge network and generates subscription links for various proxy clients (Clash, Surge, Sing-box, V2Ray, Shadowrocket, etc.).

Based on [zizifn/edgetunnel](https://github.com/zizifn/edgetunnel).

## Repository Structure

- **`明文源吗`** — The cleartext/unobfuscated JavaScript source (~414KB single file). This is where all development happens.
- **`少年你相信光吗`** — The obfuscated production build (~1.6MB). Auto-generated by CI; never edit directly.
- **`snippets`** — A standalone deployment snippet with hardcoded config values (UUID, proxy settings). Used as an alternative entry point.
- **`wrangler.toml`** — Cloudflare Workers config. Compatibility date must be `2026-01-20`.

## Build & Deploy Pipeline

There is no local build step. Everything runs through GitHub Actions:

- **Obfuscation** (`.github/workflows/obfuscate.yml`): Triggered on push to any branch when `明文源吗` changes. Runs `javascript-obfuscator` on the source and commits the result as `少年你相信光吗`.
- **Release** (`.github/workflows/test.yml`): Triggered on tag push. Copies `少年你相信光吗` → `_worker.js`, bundles it with `wrangler.toml` into `Pages.zip`, and creates a GitHub Release.

To deploy: push a git tag. The release ZIP can be uploaded to Cloudflare Pages or deployed as a Worker.

## Code Architecture

The entire application is a single JavaScript file (`明文源吗`) that exports a Cloudflare Worker `fetch` handler. All variable names and function names are in Chinese.

Key concepts:

- **Authentication**: UUID-based token (`认证令牌`) validated on every request. Configurable via KV or hardcoded.
- **Protocol handling**: VLESS and Trojan protocol parsing happens via WebSocket upgrade. The worker decodes protocol headers, extracts destination address/port, and proxies traffic through `cloudflare:sockets`.
- **Region detection** (`检测工作器地区`): Auto-detects the worker's data center region from `request.cf.country` with a country-to-region mapping table. Falls back to HK. Used to select optimal proxy IPs.
- **IP optimization**: Fetches best IPs from GitHub repos and external APIs on a 15-minute cycle with caching.
- **Subscription generation**: Detects client type from User-Agent and returns protocol-specific subscription configs (Clash YAML, Sing-box JSON, V2Ray base64, etc.).
- **Management UI**: HTML-based admin panel served at the root path (when authenticated). KV-backed configuration that takes effect immediately without redeployment.
- **Multi-language**: Chinese and Farsi, auto-detected from browser language.

## Configuration

Settings can be provided via:
1. Hardcoded constants at the top of the source file
2. Cloudflare KV bindings (runtime, managed through the web UI)
3. Environment variables in `wrangler.toml`

Key config constants in the source:
- `认证令牌` — Auth UUID token
- `回退地址` — Fallback proxy address
- `代理5配置` — SOCKS5 proxy config
- `手动工作器地区` — Manual region override (US, SG, JP, HK, KR, DE, SE, NL, FI, GB)
- `自定义路径` — Custom URL path (replaces UUID-based path)
- `仓库优选网址` — GitHub URL for best-IP list
- `启用仓库优选` / `启用其他优选` — Toggle IP optimization sources

## Development Notes

- The source uses Chinese identifiers throughout (e.g., `解码64` = base64 decode, `安静关闭套接字` = quietly close socket, `格式化标识` = format UUID).
- The `snippets` file is a self-contained version with inline config — useful for quick single-file deployment without KV.
- Obfuscation settings include: string array encoding (base64), control flow flattening, dead code injection, and self-defending (see `obfuscate.yml` for full options).
- Cloudflare compatibility date `2026-01-20` is required — the worker uses `cloudflare:sockets` API which depends on this.
